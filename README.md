# MVP Control Panel MCP

Local MCP server and control panel for turning product ideas into fast MVP builds with coordinated Codex and Claude subagents.

The project provides:

- MCP tools for project/spec/task/risk/decision/release management.
- SQLite-backed project state.
- Local browser UI with Simple and Advanced modes.
- Runner orchestration for routed subagents.
- Automatic fallback between Codex and Claude when an agent hits a token, context, session, rate, or quota limit.

## Current Agent Routing

- Backend: Claude
- Frontend: Claude, with fallback to Codex on resource limits
- QA: Codex, with fallback to Claude on resource limits
- Docs: Codex, with fallback to Claude on resource limits

Generated MVP product files are written outside this MCP repository by default:

```txt
C:\Users\FSOS\mvp-projects
```

Override with:

```bash
MVP_PROJECTS_DIR=/path/to/mvp-projects
```

## Requirements

- Node.js with `node:sqlite` support
- npm
- Codex CLI for Codex-backed tasks
- Claude Code CLI for Claude-backed tasks

## Install

```bash
npm install
npm run build
```

## Run MCP Server

```bash
npm start
```

The MCP server entrypoint is:

```txt
build/index.js
```

## Run Local UI

```bash
npm run ui
```

Open:

```txt
http://127.0.0.1:4173
```

Simple mode is intended for normal use: describe an idea, create the MVP, and watch agent progress.

Advanced mode exposes specs, tasks, workflows, subagent briefs, runner commands, decisions, risks, and release notes.

## Run Agent Runner

Run one routed task:

```bash
npm run runner -- --project-id <projectId> --provider auto --once
```

Run until idle:

```bash
npm run runner -- --project-id <projectId> --provider auto --stop-when-idle --max-cycles 8
```

Dry-run routing:

```bash
npm run runner -- --project-id <projectId> --provider auto --once --dry-run
```

## MCP Client Config Example

Use the absolute path to this repository after cloning/building.

```json
{
  "mcpServers": {
    "mvp-control-panel": {
      "command": "node",
      "args": ["C:/path/to/mcp-server/build/index.js"]
    }
  }
}
```

## Notes

- Runtime SQLite data is intentionally ignored by git.
- `build/` is generated by `npm run build` and is intentionally ignored.
- Local Claude settings are ignored.
